# Наташа · кабинет преподавателя танцев

Мини-CRM для преподавателя танцев. Ядро — **карточка клиента** и **управление абонементами**,
а не воронка продаж. Главный экран показывает, кому пора продлевать и кто пропал.

**Статус:** в продакшене, преподаватель вносит реальные данные.
База — Supabase (Postgres), хостинг — Vercel или собственный сервер
(см. [ПЕРЕНОС-НА-СЕРВЕР.md](ПЕРЕНОС-НА-СЕРВЕР.md)).

## Возможности

- **Дашборд** — «Требуют внимания» (мало занятий / скоро срок / был на пробном / пропал /
  отходил 2 абонемента, но не купил тренажёр), занятия на сегодня, финансы месяца и прогноз выручки.
- **Клиенты** — список с фильтром по статусу и поиском; карточка с контактами, источником лида
  (+ «от кого»: блогер/канал/кампания), статистикой (посещения, сумма покупок, последнее занятие),
  целями, заметками, рекомендациями для преподавателя, отметкой тренажёра и удалением.
- **Абонементы** — любое число занятий (по умолчанию 4), онлайн/офлайн, истечение по занятиям или
  по сроку, заморозка с датой и причиной. На странице абонемента: календарь посещений (отметка по дням),
  ручная правка использованных занятий, история. Остаток и статус считаются автоматически.
- **Разовые и пробные визиты** — без абонемента, оплата за визит (пробное онлайн 500 / офлайн 1000,
  разовое 2300). Считаются в посещениях и в доходах.
- **Тренажёр** — отметка покупки в карточке; напоминание предложить тем, кто отходил 2 абонемента.
  Прибыль преподавателя с продажи — 3000 ₽.
- **Занятия и посещаемость** — создание занятия, запись клиентов, отметка «была / не была».
  «Была» списывает 1 занятие с активного абонемента нужного типа; без абонемента — как разовое.
- **Финансы** — по месяцам (переключатель ‹ месяц ›): выручка (абонементы + разовые/пробные +
  прибыль с тренажёров), расходы по категориям (аренда зала, оборудование, реклама… можно вписать свою),
  прибыль. Отдельный блок «Сейчас»: средний чек, активные, ждут продления, прогноз.
- **Источники лидов** — аналитика за период: сколько лидов и сколько из них дошли до покупки
  по каждому источнику и по каждому блогеру.
- **Вход по паролю** — одна учётка преподавателя, сессия в подписанной куке, помнит вход год.

## Стек

- Next.js 16 (App Router, Turbopack) + React 19 + TypeScript
- Tailwind CSS v4
- Prisma 7 + PostgreSQL через driver adapter `@prisma/adapter-pg`
- База — Supabase (Session pooler), хостинг — Vercel (`git push` → авто-деплой)
- Авторизация — своя, на HMAC-подписи (Web Crypto, edge-совместимая)
- Шрифт Manrope (кириллица), mobile-first

## Данные (модели)

`Client`, `Subscription`, `SubscriptionVisit`, `SingleVisit`, `Lesson`, `Attendance`,
`Note`, `ClientGoal`, `Expense`. Полная схема — [`prisma/schema.prisma`](prisma/schema.prisma).
Статусы хранятся строками; «живой» статус абонемента (активен / скоро / закончился) и клиента
считается на лету в [`src/lib/domain.ts`](src/lib/domain.ts).

## Переменные окружения

| Переменная | Зачем | Где взять |
|---|---|---|
| `DATABASE_URL` | подключение к Postgres | Supabase → **Connect → Session pooler** (порт 5432) |
| `APP_PASSWORD` | пароль входа преподавателя | задаёте сами |
| `AUTH_SECRET` | подпись сессии | `openssl rand -hex 32` |

Локально лежат в `.env` (`DATABASE_URL`) и `.env.local` (`APP_PASSWORD`, `AUTH_SECRET`).
Эти файлы **не в git** (секреты). Шаблон со значениями-заглушками — [`.env.example`](.env.example).

## Запуск локально

```bash
npm install
npm run dev        # http://localhost:3000
```

База уже в облаке (Supabase) — поднимать локально ничего не нужно.
Двойной клик по файлу **«Запустить кабинет.command»** делает то же самое и открывает браузер.

## Перенос на сервер

Пошаговый гайд: [**ПЕРЕНОС-НА-СЕРВЕР.md**](ПЕРЕНОС-НА-СЕРВЕР.md).

## Команды

| Команда | Что делает |
|---|---|
| `npm run dev` | dev-сервер |
| `npm run build` | продакшн-сборка |
| `npm run start` | запуск собранной версии |
| `npm run db:push` | применить изменения схемы к базе |
| `npm run db:studio` | визуальный редактор базы (Prisma Studio) |

> ⚠️ **`npm run db:seed` и `npm run db:reset` НЕ запускать на проде** — они затирают базу
> тестовыми данными. Сейчас в базе реальные клиенты Наташи.

## Структура

```
prisma/schema.prisma     модель данных
src/lib/db.ts            подключение к Postgres (adapter-pg)
src/lib/domain.ts        бизнес-логика (статусы, остатки, цены, напоминания)
src/lib/queries.ts       запросы к БД (дашборд, списки)
src/lib/actions.ts       server actions (создание/изменение)
src/lib/auth.ts          подпись сессии; auth-actions.ts — вход/выход
src/middleware.ts        проверка входа на каждый запрос
src/app/                 страницы (дашборд, клиенты, занятия, абонементы, финансы, источники, вход)
src/components/          UI-компоненты
```
